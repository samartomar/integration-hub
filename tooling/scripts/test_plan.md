# Integration Hub POC – Test Plan

Run `tooling/scripts/reset_and_seed.sql` (or `./tooling/scripts/aws_reset_and_seed.sh`) before each test run for a clean baseline.

**Environment:** Generate Postman first: `python tooling/scripts/generate_postman.py`. Import `postman/Integration-Hub-POC.postman_environment.json`. Set `baseUrlVendorApi` and `baseUrlAdminApi` from CDK outputs (`VendorApiInvokeUrl`, `AdminApiInvokeUrl`), or run `.\tooling\scripts\get-postman-urls.ps1 -Update`.

---

## 1. Happy Path

| Step | Request | Expected |
|------|---------|---------|
| 1 | **Register Vendor** (Onboarding) – POST `{{baseUrlVendorApi}}/v1/onboarding/register` | 200. Save `apiKey` into `vendorApiKey` env var. |
| 2 | **Execute Integration** – POST `{{baseUrlVendorApi}}/v1/integrations/execute` with Authorization: Bearer &lt;JWT&gt;, body: targetVendor=LH002, operation=GET_RECEIPT, parameters.transactionId=tx-happy-1 | 200. `transactionId`, `correlationId`, `status: completed`, `downstream` (postman-echo response). |
| 3 | **List Transactions** – GET `{{baseUrlAdminApi}}/v1/audit/transactions?vendorCode=LH001&limit=10` | 200. `transactions` contains the Execute transaction. |
| 4 | **Get Transaction by ID** – GET `{{baseUrlAdminApi}}/v1/audit/transactions/{{transactionId}}?vendorCode=LH001` | 200. Full transaction details, `request_body` includes `sourceVendor` (derived). |

---

## 2. Replay (Idempotency)

| Step | Request | Expected |
|------|---------|---------|
| 1 | Set `idempotencyKey` = `replay-test-001` (or leave empty to auto-gen once). | — |
| 2 | **Execute Integration** – same body, same idempotencyKey | 200. `replayed: false`, first response. |
| 3 | **Execute Integration** – same request again | 200. `replayed: true`, same `transactionId`/`correlationId` as step 2. No new downstream call. |

---

## 3. Validation Failure

| Step | Request | Expected |
|------|---------|---------|
| 1 | **Execute Integration** – omit `targetVendor`, `operation`, or `parameters` | 400. `VALIDATION_ERROR`, `targetVendor`/`operation` is required. |
| 2 | **Execute Integration** – invalid schema: `parameters: {}` (transactionId missing) | 400. `VALIDATION_ERROR`, schema violation for `transactionId` (minLength 1). |
| 3 | **Execute Integration** – no Authorization header | 401. `AUTH_ERROR`, Missing Authorization. |
| 4 | **Execute Integration** – invalid/expired JWT | 401. `AUTH_ERROR`. |

---

## 4. Redrive

| Step | Request | Expected |
|------|---------|---------|
| 1 | Create a **failed** transaction (e.g. temporary downstream failure, or force one via DB/manual). | Get `transactionId` of a failed/downstream_failed transaction. |
| 2 | **Redrive** – POST `{{baseUrlAdminApi}}/v1/admin/redrive/{{transactionIdForRedrive}}` | 200. New transaction, `redriveOf` references original. |

---

## 5. AI Tool – NEEDS_INPUT + Valid Run

| Step | Request | Expected |
|------|---------|---------|
| 1 | Invoke Bedrock Agent with **partial** input (e.g. missing `parameters` or `transactionId`). | Agent returns `NEEDS_INPUT` or similar, asking for missing fields. |
| 2 | Invoke Bedrock Agent with **full** input: sourceVendor, targetVendor, operationCode=GET_RECEIPT, parameters.transactionId=tx-ai-1. | 200. ExecuteIntegration succeeds, downstream (postman-echo) response returned. |

---

## Postman Collection Variables

| Variable | Description |
|----------|-------------|
| `baseUrlVendorApi` | Vendor REST API URL (execute, onboarding) |
| `baseUrlAdminApi` | Admin HTTP API URL (audit, registry, redrive) |
| `vendorApiKey` | API key from Register Vendor (required for Execute) |
| `idempotencyKey` | Auto-generated by pre-request script when empty |
| `targetVendor` | LH002 |
| `operation` | GET_RECEIPT |
| `vendorCode` | LH001 (for audit) |
| `transactionIdForRedrive` | Set from failed transaction for Redrive test |
